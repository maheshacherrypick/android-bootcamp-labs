Lab 21.0 – Testing

Objective

Add unit tests to the Yamba application.

Overview

In this lab, we will cover the following concepts:

  • Creating a basic unit test

  • Running unit tests

Android uses the jUnit 3 framework for unit testing.

Steps

  1. Create unit test for YambaClient getTimeline

  2. Create unit test for StatusActivity

  3. Access YambaApplication

Detailed Steps

  1. Create the directory to contain unit tests.

    1. Inside the module you want to test, navigate to the "src" directory.

    2. Create a new directory called "androidTest". This will be at the same level as the "main" directory.

    3. Inside the "androidTest" directory create a directory called "java". This is where the unit test code will be placed.

    4. Inside "java" create a package with the same value as in your "main/java" directory. Use the package wizard.

      Using the same package name is not required but ensures that the test classes have the save "visibility" as the classes under test.

  2. Create a unit test. Unit tests will be placed in the java package under the "androidTest" directory. Unit tests are created as java code.

    1. In src/androidTest/java/com.thenewcircle.yamba create a java class named "SimpleAndroidTest"

      Select "New → Java Class". There is not a wizard for unit tests in Android Studio.

    2. Extend the AndroidTestCase class 

      public class SimpleAndroidTest extends AndroidTestCase {
}

    3. And a method call testGetTimeline 

      public void testGetTimeline() {
                fail("Not yet implemented");
}
      This follows the "fail first" strategy of unit testing

  3. Run the unit test and verify that it fails. You must create a run configuration to run the test.

    1. Select "Run → Edit Configurations" from the top level menu.

    2. Select "+" to add a new run configuration and select "Android Tests"

  4. Add Android Test

    1. Define the test by entering the following values in the wizard:

  5. Create a working test

    1. Select "Run → Edit Configurations" from the top level menu. 

      public void testGetTimeline() {

    2. Run "SimpleAndroidTest" again using the play button.

      Under the "Run" tab at the bottom of the screen you should see the jRunner output:

      Runner Output
      This time you should see a green bar to signify that the test worked (Green is clean!!!, Read is dead!!!)

  6. Create a unit test for activity UI. Although you’ve tested the event service directly, as you should also verify that the UI view element that the user sees contains the correct value. Create a unit test to run the actual activity and inspect the UI element.

    1. Create a new jUnit Test Case for the StatusActivity called “StatusActivityTest”. Extend the activity testing class from: 

      public class StatusActivityTest extends
    android.test.ActivityInstrumentationTestCase2<StatusActivity>

    2. Create a new constructor 

      public StatusActivityTest() {
   super("com.thenewcircle.yamba", StatusActivity.class);
}

    3. Create setup method and an instance variable for the activity. 

      StatusActivity statusActivity;

@Override
protected void setUp() throws Exception {
   super.setUp();
   statusActivity = getActivity();
}

    4. Create a new test method called “testActivityUI” which will lookup the text element and check it’s value.

      textActivityUI method
      public void testActivityUI() {
   TextView count = (TextView) StatusActivity.findViewById(R.id.count);
   assertEquals("140", textView.getText().toString());
}

    5. Create a new run configuration and un the test until you get a green bar.

  7. Code a test application that interacts with a View of the application under test must run in the main application’s thread, also known as the UI thread. The Android testing framework provides an annotation for ensuring that code runs on the main UI thread. Create the following test method to verify that you can change the value of a UI view:

    1. Create a new method called testView and test the value of a view element.

      Use the following only as a template - your code will probably be different.

      @UiThreadTest
public void testView() {
    TextView status = (TextView) StatusActivity.findViewById(R.id.status);
    TextView count = (TextView) StatusActivity.findViewById(R.id.count);
    assertEquals("140", count.getText().toString());
    status.setText("abc");
    assertEquals("137", count.getText().toString());
}

    2. Run the test until you get a green bar.
